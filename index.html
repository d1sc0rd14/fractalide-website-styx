Vision

The vision is to create three foundational applications that run on an existing programming platform called Fractalide.

  1. Hyperflow: A Rapid Application Development (RAD) platform
  2. Fractalmarket: An integrated app marketplace
  3. Etherflow: A new Ethereum Classic client
Hyperflow facilitates the creation of applications which can be published on Fractalmarket. Etherflow allows people to purchase applications on Fractalmarket or interact with other Ethereum Classic holders and contracts.
Hyperflow: Rapid Application Development platform
Hyperflow draws much inspiration from HyperCard. HyperCard was intuitive, semantically clear and easy to learn such that non-programmers could create powerful programs that solved real world problems at hand.
HyperCard had two modes; an execution mode and development mode. Switching between the two modes was trivial and didn't require a compilation step. In develop mode, people built their Graphical User Interfaces (GUI) by dragging and dropping GUI widgets such as a button or textbox onto a canvas. This canvas was referred to as a Card. One could then write an English like programming language (called HyperTalk) to drive the execution of your Card GUI components and link Cards together. A set of Cards formed a Stack. The nearest analogy would be a Card is a web page and a Stack would be a website. People created Stacks which did anything from computer games to bookkeeping.

Hyperflow takes the good parts of HyperCard such as the simple intuitive development/execution environment that allows non-programmers to rapidly build applications, but chooses to reject certain shortcomings, namely not being able to share stacks over a network, and the English like language HyperTalk wasn't powerful enough for professional programmers.

We will swap out HyperTalk for a component oriented programming language called Flowscript (which is an implementation of a Flow-based Programming language). These components are typically written in modern day programming languages and are designed to be shareable, reusable and reproducible. The nearest analogy is Lego. Programmers create the lego blocks and non-programmers use the Lego blocks. So in this environment we can create a synergy between programmers and non-programmers. Programmers retain their custom powerful development environments and non-programmers can easily use their components to build powerful applications to solve real world problems.

Which leads us to the next point. We need a way to share these powerful components with the world!
Fractalmarket: an Integrated App Marketplace
Once the above HyperCard layer is completed applications can be built. A Marketplace app will be among the first few applications created. Very similar to how you have an Appstore app on your phone, this Fractalide Marketplace app allows you to search and pay for applications that run on Fractalide.

Though unlike the appstore, components that make up an application are shareable and may be included in other component hierachies. The original author is compensated for their work every time their component is sold even when it forms part of a component hierarchy created by another person.

These dynamics reward programmers for creating intuitive, reusable, efficient, well maintained components.

The last notable point about the marketplace is that a new cryptocurrency will be created enabling trade between buyers and sellers. This is the very same coin that can be purchased in the ICO.
Etherflow: A New Ethereum Classic Client
Etherflow builds on top of the work of ETCDEV by utilizing the modular Ethereum Classic Emerald SDK libraries. Etherflow aims to be a fully singing and dancing Ethereum Classic client. Allowing applications created by Hyperflow to perform transactions on the blockchain, interact with deployed cryptocontracts or just be wallet for the normal user.
Programmers will be hired to assist in the implementation of Emerald SDK and integration with Etherflow.

Future Work
Once the above requirements are implemented, future work on network overlays and building out libraries for the Internet of Things takes priority.

  • Programmers write components, non-programmers may compose components and sell them in the integrated marketplace. This Rapid Application Development environment is designed for non-programmers to solve problems. So money doesn't just flow to programmers.
